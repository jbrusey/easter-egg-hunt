#+title: Easter Egg Hunt

* Introduction

This is the way we run easter egg hunts in my family.
There is code - but it is not a web-site.
Instead, the code produces LaTeX output that can be printed and the prints are then cut up, stuck to individual eggs, and the eggs deposited in some location in the house.

* Prerequisites

You need the following installed before building the question set:

- Python 3.10+
- =uv=
- Python packages:
  - =pandas=
  - =pytest=
- =pandoc=
- A TeX distribution with the XeLaTeX engine (=xelatex=)

A quick default setup with =uv=:

#+begin_src bash
uv venv
source .venv/bin/activate
uv pip install pandas pytest
#+end_src

* Inputs and expected formats

The build consumes a markdown question bank and CSV location files.

* Repository layout

- =src/=: Python logic (=parse_markdown.py=)
- =data/questions/=: hunter question markdown files
- =data/locations/=: CSV location inputs
- =assets/images/=: image assets moved from legacy =Pictures/*=
- =output/=: generated =questions.md=, =questions.tex=, =questions.pdf=
- =tests/=: parser tests
- =archive/=: legacy or obsolete historical files

** Question markdown format

Each question must be a numbered top-level item, followed by exactly three indented numbered answers.

Example:

#+begin_example
1. What color are most grass lawns?
   1. Red
   2. Green
   3. Blue
2. Which animal says "meow"?
   1. Cat
   2. Dog
   3. Cow
#+end_example

Rules:
- Question lines are top-level numbered items (e.g. =1. ...=).
- Answer lines are indented and numbered (e.g. three spaces + =1. ...=).
- Each question must have exactly 3 answers.

** CSV schema for locations

Both =data/locations/locations.csv= and =data/locations/redherringlocations.csv= must use this exact header:

#+begin_example
Room,Place
#+end_example

Expected format:
- Two columns only: =Room= and =Place=
- No partial-null rows (for example, a room with an empty place, or vice versa)
- UTF-8 plain CSV

* Build commands

Run the following exact make targets:

#+begin_src bash
make output/questions.md
make output/questions.pdf
make output/questions.tex
#+end_src

What they produce:
- =make output/questions.md=: generated normalized markdown questions file
- =make output/questions.tex=: generated LaTeX output
- =make output/questions.pdf=: final printable PDF

** Configure hunters explicitly

The build now passes hunter and input settings explicitly via CLI arguments in the =Makefile=.

Default hunters:
- =Iyra=
- =Ezra=
- =Sascha=

To add a hunter:
1. Create a question file named =data/questions/<hunter-lowercase>_questions.md=
   (example: =data/questions/alex_questions.md=).
2. Update =HUNTERS= in the =Makefile= to include the new name, for example:

#+begin_src makefile
HUNTERS := Iyra Ezra Sascha Alex
#+end_src

To remove a hunter:
1. Remove the hunter name from =HUNTERS= in the =Makefile=.
2. Optionally remove or archive their question file from =data/questions/=.

You can also run the parser directly without =make=:

#+begin_src bash
python3 src/parse_markdown.py \
  --hunters Iyra Ezra Sascha \
  --questions-dir data/questions \
  --locations-file data/locations/locations.csv \
  --red-herring-file data/locations/redherringlocations.csv \
  --nqs 12 \
  --nas 3
#+end_src

Source-of-truth files (edit these):

* Troubleshooting

** Parse/build fails due to wrong indentation

Symptoms:
- Parser complains about malformed answers
- Questions are detected but answers are missing

Fix:
- Ensure each answer is indented under its question
- Keep numbered format for all three answers
- Use spaces consistently (avoid mixed tab/space indentation)

** Parse/build fails due to wrong number of questions

Symptoms:
- Validation or tests fail with question count mismatch

Fix:
- Confirm the source question file has the expected number of top-level question entries
- Ensure each question starts with a numbered item marker and is not accidentally nested

** Parse/build fails due to CSV partial null rows

Symptoms:
- CSV processing errors
- Missing location values during build

Fix:
- Remove rows where =Room= or =Place= is blank
- Ensure both columns are present on every non-header row
- Re-save as standard CSV and re-run the make target
